Things I Learnt Whilst Making This Site

I have tried my best to keep a list of things I’ve learnt whilst putting together this site.

It won’t have the cleanest language, and is in no particular order:

  • sublime2 plugins: Package Control, Advanced new file, sublime linter, git,
  • keyboard shortcuts: close window (⌘W), switch tab(ctrltab), select line from caret(⌘shift+left/right), split line (⌘shift+L), wrap selection with tag (CtrlShift+W), Indenting/Outdenting (⌘+[ or ])
  • you can @import CSS style sheets.
  • ../ means up-one in the file structure
  • background attachment: fixed
  • Semantically, content images should be in html not in css divs, and vice-versa
  • CSS tables/containers and table-inline
  • you can clip mask images into circles using css, other shapes prove more tricky.
  • People disagree on when to use id and class selectors, i'm not sure i've found a rule yet
  • The C in CSS is quite important but I don't really understand 'the cascade'.
  • Floats and Overflow are confusing.
  • Git is useful. Learned init, add, commit, and push, but need practice.
  • !! sudo is super useful for long terminal commands you fuck up.
  • GitHub and Knowledeable Friends are the best <3
  • flexbox looks cool but isnt that supported, also position: absolute did what I wanted after a bit of understanding and messing about
  • I need to study parent/sibling/child relationships.
  • I need better understanding of position: relative.
  • Aint nobody got time for floats.
  • Close container divs or it's pointless having them!
  • Border: solid colour is useful for layout diagnostics.
  • Seriousy, fuck floats.
  • Coding after work, past 11pm, is not productive.
  • Orientation: portrait in MediaQ's for phones with retina displays?
  • You can have -or- in mediaQ's, just seperate rules with comma. Need to do further research on syntax etc.
  • Developing for phones is hard because inspect-element doesnt't exist. Chrome and Safari display my site differently, neither displays it how I'd expect
  • text-decoration: none; takes away underline
  • list-style-type: none; takes away bullet points
  • vertical-align: is useful for making things more typographically pretty.
  • Coding dreams are wierd.
